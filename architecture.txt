go app with 4 services that fetch data from apis
parsers for each and db for each

use workers for concurrency in fetching (most relevant here), processing and storing
use scheduler to run services at specific times (cron jobs)
use channels for pipline
go routines, locking (db)

weatherapi -- use city name
openaq -- use country id
worldtimeapi -- use zone name
restcountries -- use country name

root
  cmd/app
    - main.go
  services
    weather
      fetch
      parse
      store
    aqi
    time
    country
  internal
    config
    db
    logger (can also define in main)
    scheduler
    workerpool
    channels
  models
  DockerFile
  docker-compose.yml
  .gitignore
  .env
